# 蔵書一括削除機能の実装

## 概要
蔵書リスト画面において、複数の書籍をチェックボックスで選択し、まとめて削除できるようにしました。また、フィルタにかかっている全ての書籍をまとめて選択することも可能です。

## 実装内容

### 1. フロントエンド（Vue.js）変更
**ファイル**: `resources/js/pages/BookIndex.vue`

#### 追加機能:
- **チェックボックス表示**: 各書籍カードの左側にチェックボックスを追加
- **選択状態の視覚化**: 選択された書籍は背景色が変わる（ブルーハイライト）
- **一括選択機能**:
  - フィルタ結果が0冊の場合: 「すべて選択」ボタンを表示
  - 1冊以上選択時: 「フィルタ結果をすべて選択」リンクを表示
  - フィルタにかかっている全ての書籍をまとめて選択可能
- **一括削除ツールバー**: 1冊以上選択された場合、画面上部に以下が表示されます:
  - 選択冊数の表示
  - 「フィルタ結果をすべて選択」リンク（未選択の書籍がある場合）
  - 「選択を解除」ボタン
  - 「選択した書籍を削除」ボタン
- **フィルタ連動**: フィルターやソートが変更されると、自動的に選択がクリアされます

#### 主要メソッド:
```javascript
// 書籍選択の切り替え
toggleBookSelection(book)

// フィルタ結果をすべて選択
selectAllFiltered()

// 選択をすべて解除
clearSelection()

// 選択した書籍を削除
deleteSelectedBooks()
```

#### 状態管理:
```javascript
const selectedBooks = ref(new Set());  // 選択された書籍IDのSet
const isDeleting = ref(false);          // 削除処理中フラグ
```

### 2. バックエンド（Laravel）変更
**ファイル**: `app/Http/Controllers/Api/BookController.php`

#### 新規メソッド:
```php
public function batchDestroy(Request $request)
```

このメソッド:
- 権限チェック（編集権限が必要）
- リクエストバリデーション
- 複数IDに対する一括削除処理
- エラーハンドリング

#### レスポンス:
```json
{
  "message": "XX冊の書籍を削除しました",
  "success": true,
  "deleted_count": 20,
  "failed_count": 0,
  "failed_books": []
}
```

### 3. ルート設定
**ファイル**: `routes/api.php`

```php
Route::post('books/batch-destroy', [BookController::class, 'batchDestroy']);
```

## 使用方法

### パターン1: 個別に書籍を選択して削除

1. **蔵書一覧ページを開く**
   - `/books` に移動

2. **書籍を選択**
   - 書籍カード左側のチェックボックスをクリック
   - 選択された書籍は背景がブルーハイライトされます

3. **複数選択**
   - 複数の書籍を同じように選択
   - 画面上部の一括削除ツールバーに選択冊数が表示されます

4. **一括削除**
   - ツールバーの「選択した書籍を削除」ボタンをクリック
   - 確認ダイアログが表示されます
   - 「OK」をクリックで削除を実行

### パターン2: フィルタ結果をすべて選択

1. **フィルターを適用**
   - タイトル検索、著者検索、NDC分類などでフィルター
   - 例：「NDC分類：007 プログラミング」で絞り込み

2. **「すべて選択」ボタンをクリック**
   - フィルタ結果が表示されている状態で、右側の「すべて選択」ボタンをクリック
   - フィルターにかかっている全ての書籍が選択されます

3. **一括削除**
   - ツールバーの「選択した書籍を削除」ボタンをクリック
   - 確認ダイアログが表示されます

### パターン3: 部分的な追加選択

1. **フィルタを適用してすべて選択**
   - 例：「NDC分類：007」でフィルター → 「すべて選択」

2. **さらに一部の書籍を追加選択**
   - 別フィルターで新たに表示された書籍のチェックボックスをクリック
   - 既に選択されている書籍と新たに選択した書籍がまとめて削除されます

## 主な特徴

### フィルタ対応
- タイトル検索、著者検索、NDC分類フィルターなどを使用しながら選択可能
- フィルター結果に含まれる書籍のみを選択する機能

### インテリジェントなUI
- フィルタ結果が0冊の場合：「すべて選択」ボタンを表示
- 1冊以上選択時：「フィルタ結果をすべて選択」リンクを表示（未選択の書籍がある場合）
- 全て選択済みの場合：リンクを非表示

### ユーザーフレンドリー
- 選択状態が背景色で視覚的にわかりやすい
- 選択冊数をリアルタイムで表示
- 誤操作防止の確認ダイアログ

## 権限チェック

- **管理者のみ利用可能**: 書籍編集権限が必要です
- チェックボックスはadmin権限がある場合のみ表示されます

## エラーハンドリング

- 削除に失敗した書籍がある場合は、失敗した書籍情報が返されます
- ネットワークエラーの場合はアラート表示
- 権限がない場合は403エラーが返されます

## データベース

削除の実装は既存の `Book::delete()` を使用しており、
関連テーブル（BorrowやJANCode）との外部キー制約が適切に設定されていることを前提としています。

## テスト方法

```javascript
// Tinkerでテスト
php artisan tinker
$books = Book::limit(5)->get();
$ids = $books->pluck('id')->toArray();

// APIを呼び出す
$response = App\Models\Book::whereIn('id', $ids)->delete();
```
